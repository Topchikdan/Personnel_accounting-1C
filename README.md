## О проекте

Конфигурация автоматизирует базовый кадровый учёт: ведение организационной структуры (отделы, должности), карточек сотрудников и истории их кадровых перемещений — от приёма на работу до увольнения.

Исходники выгружены в XML-формате штатным механизмом платформы (**Конфигурация → Выгрузить конфигурацию в файлы...**) и версионируются через git. Формат позволяет вести историю изменений и работает на базовых версиях платформы без хранилища конфигурации.

---

## Возможности

- Ведение карточек сотрудников
- Учёт организационной структуры: отделы и должности
- Оформление приёма на работу, увольнения и кадровых перемещений
- Хранение полной истории кадровых изменений по сотруднику
- Отчёт по текущему списку работающих сотрудников
- Просмотр организационной структуры

## Быстрый старт

```bash
# 1. Клонируйте репозиторий
git clone https://github.com/Topchikdan/Personnel_accounting-1C.git

# 2. Откройте базу в Конфигураторе 1С и выполните:
#    Конфигурация → Загрузить конфигурацию из файлов...
#    Укажите путь к папке репозитория

# 3. Обновите конфигурацию базы данных (F7)
```

## Состав конфигурации

### Справочники

| Справочник | Назначение |
|---|---|
| `Сотрудники` | Карточки сотрудников организации |
| `Должности` | Перечень должностей |
| `Отделы` | Структурные подразделения (отделы) |
| `ПолСотрудника` | Справочник пола сотрудника |

### Документы

| Документ | Назначение |
|---|---|
| `ПриемНаРаботу` | Оформление приёма сотрудника на работу |
| `Увольнение` | Оформление увольнения сотрудника |
| `КадровоеПеремещение` | Перевод сотрудника между отделами/должностями |
| `КадровыеПриказы` | Кадровые приказы по сотрудникам |

### Регистры

| Регистр | Назначение |
|---|---|
| `КадроваяИстория` | Регистр сведений, хранящий историю кадровых изменений по сотруднику (приём, перемещения, увольнение) |

### Отчёты и обработки

| Объект | Назначение |
|---|---|
| `СписокРаботающихСотрудников` | Отчёт по текущему списку работающих сотрудников |
| `ОргСтруктура` | Просмотр организационной структуры |

## Разработка

### Обновление репозитория после изменений конфигурации

1. Откройте базу в режиме **Конфигуратор**.
2. Выполните **Конфигурация → Выгрузить конфигурацию в файлы...**, указав папку репозитория — изменятся только реально изменённые файлы.
3. Зафиксируйте изменения в git:

```bash
git add .
git commit -m "Описание изменений"
git push
```

### Развёртывание конфигурации из репозитория

1. Создайте новую информационную базу в 1С (или используйте существующую).
2. В Конфигураторе выполните **Конфигурация → Загрузить конфигурацию из файлов...**, указав путь к папке репозитория.
3. Обновите конфигурацию базы данных (**F7**).
